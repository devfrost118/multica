# Не выдумывать имена метаданных и API

- Не придумывай имена справочников, документов, регистров, реквизитов,
  перечислений, общих модулей и их методов. Проверяй имя по XML-выгрузке или
  по существующему коду проекта.
- Сохраняй русскоязычные идентификаторы и проектные префиксы как есть, если
  задача явно не требует переименования.
- Не изобретай несуществующие методы и параметры платформенного API.
  Опирайся на документированные методы 1С:Предприятие и на уже принятые в
  проекте паттерны.
- Синоним и имя — разные вещи: для обращения из кода используй имя, не синоним.
