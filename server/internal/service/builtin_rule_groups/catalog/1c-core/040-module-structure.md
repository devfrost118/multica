# Структура модулей и проектные соглашения

- Размещай код в правильном модуле: модуль объекта, модуль менеджера, общий
  модуль, модуль формы — по назначению, а не «куда ближе».
- Соблюдай флаги общих модулей (Клиент/Сервер, Вызов сервера, Внешнее
  соединение, Привилегированный) — они определяют, откуда метод доступен.
- Следуй уже принятому в проекте стилю: именование процедур и переменных,
  области (`#Область`), экспортность, комментарии-описания методов.
- Не вводи параллельные абстракции, если в проекте уже есть подходящий
  механизм. Сначала ищи существующий паттерн.
