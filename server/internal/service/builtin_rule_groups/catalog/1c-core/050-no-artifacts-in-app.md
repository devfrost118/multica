# Не создавать служебные файлы внутри app/ выгрузки

Директория XML/BSL-выгрузки конфигурации/расширения (обычно `app/`) — это
строго структурированный экспорт, который загружается обратно в 1С.

- Не создавай внутри неё служебные файлы агента: заметки, планы, `reply*.md`,
  `handoff*.md`, временные скрипты.
- Не клади туда копии внешних репозиториев и markdown-инструкций.
- Любые рабочие заметки и документация проекта — в `docs/` или вне выгрузки,
  не среди объектов метаданных.

Посторонние файлы в выгрузке ломают загрузку конфигурации и засоряют diff.
