# Сначала метаданные, потом BSL

Метаданные конфигурации (XML-выгрузка) — источник истины. Прежде чем менять
код модуля:

- Найди и прочитай XML объекта (`*.xml`): реквизиты, табличные части,
  измерения/ресурсы регистров, формы, команды, макеты.
- Пойми, owned-объект расширения это или заимствованный из базовой
  конфигурации (CFE). Для заимствованных меняй только разрешённые свойства.
- Убедись, что имя объекта, реквизита и типа существует в метаданных, прежде
  чем обращаться к нему из BSL.

Менять модуль вслепую, не сверившись с метаданными, — частая причина ошибок
компиляции и расхождений с реальной структурой данных.
