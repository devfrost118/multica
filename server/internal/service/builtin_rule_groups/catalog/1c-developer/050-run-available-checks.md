# Запускай доступные проверки

После изменений прогони то, что реально доступно в окружении:

- well-formedness XML изменённых файлов выгрузки;
- BSL-диагностику / module health (BSL Language Server, EDT), если подключены;
- проектные тесты и скрипты сборки/валидации, если они есть.

Если платформа 1С / EDT / Configurator в окружении недоступны — выполни
статические проверки и **явно укажи**, что runtime-валидация не проводилась.
Не заявляй о прохождении проверки, которую не запускал.

При изменении BSL/метаданных под LSP/MCP — обнови индекс (нотификация об
изменённых файлах или переиндексация), иначе диагностика будет устаревшей.
