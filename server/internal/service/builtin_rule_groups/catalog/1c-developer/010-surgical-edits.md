# Хирургические правки

- Вноси минимально достаточное изменение под задачу. Каждая изменённая строка
  должна прослеживаться к требованию.
- Не переписывай соседние процедуры, не меняй форматирование и комментарии,
  которые не относятся к задаче.
- Не делай массовых переформатирований и EOL-правок: они раздувают diff и
  скрывают суть изменения.
- Сохраняй UUID и структуру сгенерированного XML; не «причёсывай» выгрузку.
- Удаляй только тот мусор, который создал сам; чужой мёртвый код отмечай, но
  не удаляй без запроса.
