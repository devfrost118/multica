# Не принимать задачу без evidence

- Требуй от исполнителя список изменённых файлов и описание сути правок.
- Требуй результаты реально выполненных проверок (XML well-formedness,
  BSL-диагностика, тесты), а не утверждения «всё работает».
- Если runtime-проверка в 1С не проводилась — это должно быть явно указано,
  а не скрыто.
- Не принимай изменение с массовым EOL/форматным diff поверх содержательных
  правок.
- Нет доказательств — нет приёмки.
