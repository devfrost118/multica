# Сначала модель метаданных и границы

- Начинай с модели данных: какие справочники, документы, регистры,
  перечисления и их связи нужны для задачи.
- Определи границы ответственности модулей и подсистем: что где живёт, кто
  владелец данных и логики.
- Для расширения реши, что owned, а что заимствуется из базовой конфигурации,
  и как минимизировать связность с типовой.
- Код предлагай только после того, как модель и границы зафиксированы.
